# **Task 1 - Git for Source Control and Lifecycle Management**

## **Outline**

The student must employ and demonstrate the use of an industry standard Software Development Lifecycle (SDLC) demonstrated by the effective use of Git as source control management system for the duration of your project through to submission. 

The use of Git should facilitate and evidence the following
  -	an engineering process via a decision record (commit messages) and well-structured repository.
  -	feature development for software solutions required in tasks 2 - 3 for the coursework
  -	testing and documentation
  - tags for milestones and delivery status including submission

## **Assessment Criteria**
Task 1 is worth a total of **20** marks. This is assessed according to the following four criteria:
- Consistency of commits 
- Evidence of software engineering principles 

These are discussed detailed below. The points listed describe what a high‑performing solution might typically demonstrates. They are not mandatory design choices—students should set their own goals and justify their approach. Different, well‑reasoned solutions can still achieve high marks. 

### 1. Consistency of commits
- Regular commits throughout the project.
- use of tags to represent key development milestones 
- Informative and structured commit messages
> 10 marks
### 2. Evidence of software engineering principles
- enhanced maintainability through documentation e.g. markdown or text readme files
- interpretation of requirements
- unit testing
- well structured repo
> 10 marks
