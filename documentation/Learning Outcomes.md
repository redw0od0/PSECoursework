
# Learning Outcomes

**Many of the elements in the module spec require knowledge and understanding and find direct application in this coursework**. Key learning outcomes have been `have been highlighted`.

## Knowledge and Understanding

**On successful completion of this module, students should be able to:**

1. Define basic programming concepts such as data types, variables, and operators.  
2. Explain the principles of programming structures such as decision control, loops, and the use of functions, classes, and external libraries.
3. `Describe elements of engineering approaches to software development and maintenance.`
4. Demonstrate understanding of vulnerability, security, and ethical considerations in software engineering practice.
5. Recognise different programming paradigms (imperative, procedural, object‑oriented) and their application.

---

## Subject‑Specific – Cognitive Skills

**On successful completion of this module, students should be able to:**

1. `Interpret and evaluate software code and reason why certain approaches, algorithms, or data structures may have been used.`
2. `Critically assess requirements for testing, security, documentation, and maintainability, within the software development lifecycle.`

---

## Subject‑Specific – Practical Skills

**On successful completion of this module, students should be able to:**

1. `Operate and use integrated development environments (IDEs) and version control technologies.`
2. `Interpret and derive software requirements, and write functional code and data structures to deliver solutions.`
3. `Resolve coding errors, develop practical debugging skills, conduct appropriate testing, and employ defensive programming.`

---

## Key Transferable Skills

**On successful completion of this module, students should be able to:**

1. Demonstrate aptitude for programming and apply key concepts to different or emerging software environments and languages.
2. Apply knowledge to related disciplines such as computer network systems, microcontrollers, data analytics, and data visualisation.
3. `Strengthen conceptual understanding of structured engineering approaches, verification, and validation.`
4. `Apply problem‑solving and analytical thinking to practical tasks.`
5. `Communicate effectively through clear expression of requirements and the production of high‑quality documentation.`

`Understanding and use of programming elements: data types, variables and operators`.
