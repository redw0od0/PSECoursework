## **Academic Integrity**
**Your coursework submission must be your own work.** Academic misconduct will not be tolerated. Students should be familiar with University [regulation XVIII (academic misconduct)](https://www.lboro.ac.uk/governance/regulations/18/) also detailed in the relevant section in the [student handbook](https://www.lboro.ac.uk/students/handbook/assessments/exams-information/academic-misconduct/).
 For coursework, students should pay particular attention to the section vii:

    collusion – working with another student on an assessment which is intended to be the student’s own work. This includes submitting assessed work as the student’s own of which the student is not the sole author (because of the collusion with another student), or providing material which is submitted by another student as their own.

## **Software requirements**
For the execution and delivery of this coursework you should install require

* [Git](https://git-scm.com/) - a free and open source distributed **version control system** designed to handle software projects with speed and efficiency.
* [Visual Studio Code](https://code.visualstudio.com/) (VSC) (an Integrated Development Environment (IDE).
* [Arduino IDE](https://www.arduino.cc/en/software/) - sketch editor and uploader for the Arduino
* Optional You can install arduino extension for VSCode PlatformIO IDE

### **Optional**
* [Github](https://github.com/) you may optional register a Github account and push your repo to the cloud. This is **not** required for your coursework but might be useful to

  * keep a cloud back hub of your repo
  * enable you to work on different machines seemlessly
  * extend your learning

If and when registering external accounts or for external services **do not use your university password**. 

### **Can I use another IDE?**
If you have another preferred IDE you may use it. Note however : 
* Learning materials will feature only VSC and Arduino IDE.
* We will not support other IDEs.
* Your IDE should either integrate with Git, or you should use the GiT desktop application, or a terminal command line to manage your **Repo** with Git updates and submits (see Part 1).
* You may for convenience use the arduino extension in VSC (_identifier vscode-arduino.vscode-arduino-community_). Learning materials and support however, focus on Arduino IDE.
